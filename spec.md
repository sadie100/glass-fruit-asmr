# spec.md — Glass Fruit ASMR Slicer 동작 명세

## 1. 개요

브라우저에서 유리 재질의 과일을 드래그로 슬라이스하며 ASMR 사운드를 즐기는 인터랙티브 체험 앱.

---

## 2. 화면 구성

| 요소 | 위치 | 설명 |
|------|------|------|
| 타이틀 ("Glass Fruit ASMR") | 상단 중앙 | 반투명 흰색, 대문자, 넓은 자간 |
| 점수 표시 | 타이틀 아래 | 슬라이스 횟수를 큰 숫자로 표시 |
| 안내 문구 ("드래그하여 과일을 자르세요") | 하단 중앙 (버튼 위) | 펄스 애니메이션, 첫 슬라이스 후 페이드아웃 |
| 과일 선택 버튼 | 하단 중앙 | 5개 원형 버튼 (🍎🍊🍇🍉🍓) |
| 3D 뷰포트 | 전체 화면 | 어두운 배경, 반사 바닥면, 안개 효과 |
| 커스텀 커서 | 마우스 위치 추적 | 원형 테두리, 슬라이스 중 축소+발광 |
| 슬래시 트레일 | 마우스 경로 위 | 2D 캔버스 오버레이, 시간 경과로 페이드 |

## 3. 과일 종류

| ID | 이름 | 이모지 | 크기 | 형태 특징 |
|----|------|--------|------|-----------|
| apple | 사과 | 🍎 | 1.0 | 구형 + 윗부분 움푹 + 미세 볼록, 줄기 있음 |
| orange | 오렌지 | 🍊 | 1.0 | 기본 구형 |

| strawberry | 딸기 | 🍓 | 0.7 | 뒤집힌 원뿔 + 볼록 변형, 줄기 있음 |

## 4. 유리 재질 렌더링

- **프레넬 효과**: 시선 각도에 따라 가장자리가 더 밝게 반사
- **스페큘러 하이라이트**: 2개의 방향성 광원에 의한 하이라이트 (pow 80, 60)
- **내부 발광**: UV 좌표 기반의 미세한 시간 변화 글로우
- **반투명**: 기본 opacity 0.35, 프레넬과 스페큘러에 의해 최대 0.95까지 증가
- **양면 렌더링**: DoubleSide, depthWrite 비활성

---

## 5. 슬라이스 동작

### 5.1 입력 조건
1. 포인터 다운으로 슬라이스 시작
2. 드래그 중 경로(trail)를 기록
3. 포인터 업 시 시작점과 끝점의 NDC 거리가 **0.1 초과**이어야 유효
4. 경로를 따라 레이캐스트하여 과일 메시와 **실제 교차 확인**

### 5.2 슬라이스 처리
1. 2D 슬라이스 방향에서 수직인 벡터를 계산하여 3D 절단면(Plane) 생성
2. 카메라 회전(quaternion)을 적용하여 월드 공간으로 변환
3. 원본 과일을 제거하고 **두 개의 반쪽** 생성:
   - 각 반쪽은 clippingPlane으로 잘린 유리 셰이더 메시
   - 절단면에 **단면 캡(CircleGeometry)** 부착 — 내부 색상, 링 패턴, 시머 효과
4. 유리 파편 파티클 30개 생성

### 5.3 슬라이스 후 물리
- 두 반쪽은 절단면 법선 방향으로 밀려남 (속도 1.5~2.5) + 위로 2.0
- 중력 가속도 5로 낙하
- 바닥(y = -1.3) 충돌 시 정지, 회전 감속
- 착지 후 **2초간 페이드아웃** 후 제거

### 5.4 과일 재생성
- 슬라이스 후 **2.5초 대기** 후 같은 종류의 과일 자동 재생성

---

## 6. 파티클 시스템

- **형태**: Tetrahedron 또는 Box (랜덤), 크기 0.02~0.08
- **재질**: MeshPhysicalMaterial, transmission 0.5 (반투명 유리)
- **물리**: 랜덤 방향 속도, 중력 가속도 6, 랜덤 회전
- **바닥 충돌**: y < -1.4에서 착지, 착지 사운드 재생
- **수명**: decay 0.3~0.8, opacity가 0이 되면 제거

---

## 7. ASMR 오디오

### 7.1 초기화
- 첫 포인터 다운 이벤트에서 AudioContext 초기화 (브라우저 자동재생 정책 대응)
- 컨볼루션 리버브 (3초 임펄스 응답) 생성, 리버브 게인 0.3

### 7.2 슬라이스 사운드 (3레이어)

| 레이어 | 설명 | 주요 파라미터 |
|--------|------|---------------|
| Glass Slice | 밴드패스 필터된 노이즈 — 유리 긁힘 | baseFreq → baseFreq×1.5 스윕, Q = 5~20 |
| Crystal Resonance | 4개 사인파 하모닉스 — 크리스탈 벨 | 0.5×, 1×, 1.5×, 2.2× baseFreq |
| Shard Tinkle | 3~7개의 짧은 고음 사인파 — 파편 소리 | 1.5~3.5× baseFreq, 랜덤 딜레이 |

### 7.3 과일별 오디오 특성

| 과일 | baseFreq | resonance | 특징 |
|------|----------|-----------|------|
| apple | 2200 Hz | 0.7 | 중고음, 보통 울림 |
| orange | 1800 Hz | 0.5 | 중음, 짧은 울림 |
| grape | 3200 Hz | 0.9 | 고음, 긴 울림 |
| watermelon | 800 Hz | 0.3 | 저음, 짧은 울림 |
| strawberry | 2800 Hz | 0.8 | 고음, 긴 울림 |

### 7.4 기타 사운드
- **파편 착지음**: 3000~7000 Hz 사인파, 지속 30~80ms
- **앰비언트**: 60 Hz 사인파, 게인 0.008 (상시)

---

## 8. 카메라 및 씬

- **카메라**: PerspectiveCamera, FOV 55, 초기 위치 (0, 1.5, 5)
- **OrbitControls**: 댐핑 0.05, 팬 비활성, 거리 2~10 제한, 슬라이스 중 비활성
- **조명**: Ambient(0x223344) + Directional×2 + PointLight(rim)
- **바닥**: 반사성 어두운 평면 (roughness 0.15, metalness 0.8)
- **안개**: FogExp2, density 0.08
- **톤매핑**: ACESFilmic, exposure 1.2

---

## 9. 과일 애니메이션 (대기 상태)

- Y축 회전: 초당 0.3 라디안
- 부유 효과: `sin(elapsed × 0.8) × 0.15`로 상하 이동
- 셰이더 uTime 유니폼 실시간 업데이트

---

## 10. 과일 선택 UI

- 5개 원형 버튼, 하단 중앙 가로 배치
- 호버 / 활성 상태: 테두리 밝아짐 + scale 1.1
- 클릭 시 즉시 해당 과일 생성 (기존 과일 및 잔해 제거)
- backdrop-filter blur 적용

---

## 11. 반응형 처리

- 윈도우 리사이즈 시: 카메라 종횡비, 렌더러 크기, 슬래시 캔버스 크기 동기화
- pixelRatio: devicePixelRatio의 최대 2배 제한

---

## 12. 슬래시 트레일 시각 효과

- 2D 캔버스 오버레이 (포인터 이벤트 패스스루)
- 각 세그먼트의 나이(age)에 따라:
  - 300ms 이내만 렌더링
  - opacity: `(1 - age) × 0.8`
  - 선 굵기: `(1 - age) × 4 + 1`
- 글로우 레이어: 추가 8px 두께, opacity ×0.3
- 색상: rgba(255, 220, 220) / 글로우 rgba(255, 150, 150)
