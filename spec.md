# spec.md — Glass Fruit ASMR Slicer 동작 명세

## 1. 개요

브라우저에서 유리 재질의 과일을 드래그로 슬라이스하며 ASMR 사운드를 즐기는 인터랙티브 체험 앱.

### 1.1 ASMR 유리과일 영상 레퍼런스

실제 ASMR 유리과일 영상의 전형적 특징을 디자인 기준으로 삼는다.

**슬라이싱 동작**
- 칼로 천천히, 일정한 속도로 과일을 위→아래 방향으로 압착하듯 절단
- 한 번에 깔끔하게 두 조각으로 분리 — 급격한 움직임 없음
- 자른 후 두 조각이 천천히 좌우로 벌어지며 분리되는 연출

**시각적 특징**
- 절단면이 드러나면서 내부의 투명/반투명 유리 질감 노출
- 단면에 실제 과일 내부 구조(씨, 과육 패턴 등)가 유리 재질로 표현
- 작은 유리 파편/부스러기가 절단면 주변에서 천천히 흩어짐
- 빛이 단면을 통과하면서 반짝이는 굴절/프리즘 효과
- 배경은 어둡고 조명은 과일에 집중 → 유리 질감 극대화

**소리 특징**
- 절단 시 "사각사각" 또는 "쩍" 하는 유리/젤리 느낌의 소리
- 분리될 때 "딱" 하는 짧은 크랙 사운드
- 파편이 떨어질 때 "찰랑찰랑" 하는 고음의 유리 틴클 사운드
- 전체적으로 크리스탈처럼 맑고 선명한 고주파 사운드
- 두드리면 "톡톡" 하는 유리 특유의 공명음

**연출**
- 자른 후 단면을 카메라에 가까이 보여주는 줌인
- 조각을 두드리거나 만지며 추가 사운드 생성
- 전체적으로 느리고 차분한 페이스 — ASMR의 핵심은 "급하지 않음"

## 2. 체험 흐름 (Experience Flow)

유리 과일 ASMR 영상의 전형적인 흐름을 인터랙티브하게 재현한다.

| 단계 | 행동 | 설명 |
|------|------|------|
| ① 등장 | 자동 | 유리 과일이 화면 중앙에 스케일-업 애니메이션으로 등장, 천천히 회전하며 부유 |
| ② 두드리기 | 클릭/탭 | 과일 표면을 클릭하면 유리 두드리는 소리(톡톡) 재생 + 표면 파문 이펙트 |
| ③ 슬라이스 | 드래그 | 과일을 가로질러 드래그하면 반으로 절단, 유리 절단음 + 크리스탈 공명 |
| ④ 단면 감상 | 자동 | 슬라이스 직후 카메라가 단면 쪽으로 살짝 줌인, 단면의 보석 같은 질감 강조 |
| ⑤ 추가 자르기 | 드래그 | 잘린 반쪽을 다시 슬라이스 가능 (최대 2회 추가 절단), 더 작은 조각 생성 |
| ⑥ 파편 | 자동 | 유리 파편이 절단면 부근에서 천천히 흩어지며 잔잔한 틴클 사운드 |
| ⑦ 다음 과일 | 수동 | 과일 선택 버튼으로 다른 과일 전환 시 기존 조각 정리 후 새 과일 등장 |

- 사용자는 ②~⑤를 자유롭게 반복하며 자신만의 리듬을 만든다.
- 과일 선택 버튼으로 언제든 다른 과일로 전환 가능.

## 5. 유리 재질 렌더링

- **프레넬 효과**: 시선 각도에 따라 가장자리가 더 밝게 반사
- **스페큘러 하이라이트**: 2개의 방향성 광원에 의한 하이라이트 (pow 80, 60)
- **내부 발광**: UV 좌표 기반의 미세한 시간 변화 글로우
- **반투명**: 기본 opacity 0.35, 프레넬과 스페큘러에 의해 최대 0.95까지 증가
- **양면 렌더링**: DoubleSide, depthWrite 비활성

## 6. 두드리기 (Tap) 인터랙션

### 6.1 입력
- 과일 위에서 포인터 다운 → 포인터 업, 드래그 거리 NDC **0.1 이하**이면 "탭"으로 판정

### 6.2 시각 효과
- 탭 지점에서 동심원 파문(ripple) 이펙트 — 유리 표면을 따라 퍼지며 페이드아웃
- 과일이 탭 방향으로 미세하게 찌그러졌다 복원 (스케일 펄스, 50ms)

### 6.3 사운드
- **Glass Tap**: 과일 baseFreq × 1.2 주파수의 짧은 사인파 (지속 80~120ms)
- **Resonance Tail**: baseFreq의 사인파가 resonance 값에 비례한 길이로 감쇠 (100~400ms)
- 탭마다 ±10% 랜덤 피치 변화로 반복감 방지

## 7. 슬라이스 동작

### 7.1 입력 조건
1. 포인터 다운으로 슬라이스 시작
2. 드래그 중 경로(trail)를 기록
3. 포인터 업 시 시작점과 끝점의 NDC 거리가 **0.1 초과**이어야 유효
4. 경로를 따라 레이캐스트하여 과일 메시와 **실제 교차 확인**

### 7.2 슬라이스 처리
1. 2D 슬라이스 방향에서 수직인 벡터를 계산하여 3D 절단면(Plane) 생성
2. 카메라 회전(quaternion)을 적용하여 월드 공간으로 변환
3. **절단면 위치**: 드래그 경로의 중점을 레이캐스트하여 과일 표면 교차점을 구하고, 해당 지점을 절단면이 지나도록 배치 — 사용자가 드래그한 위치에서 실제로 잘림 (과일 중심을 항상 관통하지 않음)
4. 원본 과일을 제거하고 **두 개의 조각** 생성:
   - 각 조각은 clippingPlane으로 잘린 유리 셰이더 메시
   - 절단면에 **단면 캡(CircleGeometry)** 부착 — 내부 색상, 링 패턴, 시머 효과
   - 단면은 해당 과일의 내부 구조(씨, 과육 패턴 등)를 유리 재질로 표현
   - 빛이 단면을 통과할 때 굴절/프리즘 효과로 반짝임 연출
   - 캡 위치와 크기는 실제 절단 지점에 맞게 조정
5. 유리 파편 파티클 30개 생성

### 7.3 추가 자르기 (Re-slice)
- 벌어진 조각은 계속 슬라이스 대상으로 유지
- **횟수 제한 없음** — 조각이 남아 있는 한 계속 자를 수 있음
- 추가 절단 시 파티클 수 감소 (15개), 사운드 볼륨 70%
- 조각이 일정 크기 이하이면 슬라이스 불가

### 7.4 슬라이스 후 움직임
- ASMR 컨셉에 맞게 **느리고 조용한 벌어짐** 연출
- 두 조각은 절단면 법선 방향으로 **천천히 벌어짐** (속도 0.3~0.5) — 튀어오르지 않음, 위로 솟지 않음
- 벌어지는 동안 **감속(ease-out)** 적용, 약 0.5초에 걸쳐 정지
- 벌어진 상태로 **제자리에 머무름** (낙하 없음, 바닥 충돌 없음)
- 조각은 **제거되지 않고 남아 있음** — 추가 슬라이스 대상으로 유지
- 사용자가 다른 과일을 선택하거나 과일 재생성 시에만 기존 조각 정리

### 7.5 단면 줌인 연출
- 첫 슬라이스 직후 카메라가 단면 방향으로 **0.5초간 부드럽게 줌인** (FOV 55→45)
- 1초 유지 후 **0.5초간 원래 FOV로 복귀**
- OrbitControls 일시 비활성 (줌인 연출 동안)
- 추가 자르기에는 줌인 미적용

### 7.6 과일 재생성
- 과일 선택 버튼 클릭 시 기존 조각 모두 정리 후 새 과일 생성
- 스케일 0 → 1 등장 애니메이션 (0.4초, ease-out)

## 8. 파티클 시스템

- **형태**: Tetrahedron 또는 Box (랜덤), 크기 0.02~0.08
- **재질**: MeshPhysicalMaterial, transmission 0.5 (반투명 유리)
- **물리**: 절단면 부근에서 **천천히 흩어짐** (속도 0.3~0.8), 중력 가속도 1.5, 느린 랜덤 회전
- **움직임**: 절단면에서 바깥으로 부드럽게 퍼지며 감속, ASMR 분위기에 맞는 차분한 연출
- **수명**: 2~3초간 서서히 페이드아웃, opacity가 0이 되면 제거
- **착지 사운드**: 아주 작은 틴클 사운드 (게인 0.03 이하)

## 9. ASMR 오디오

### 9.1 초기화
- 첫 포인터 다운 이벤트에서 AudioContext 초기화 (브라우저 자동재생 정책 대응)
- 컨볼루션 리버브 (3초 임펄스 응답) 생성, 리버브 게인 0.3

### 9.2 슬라이스 사운드 (3레이어)

| 레이어 | 설명 | 주요 파라미터 |
|--------|------|---------------|
| Glass Slice | 밴드패스 필터된 노이즈 — 유리 긁힘 | baseFreq → baseFreq×1.5 스윕, Q = 5~20 |
| Crystal Resonance | 4개 사인파 하모닉스 — 크리스탈 벨 | 0.5×, 1×, 1.5×, 2.2× baseFreq |
| Shard Tinkle | 3~7개의 짧은 고음 사인파 — 파편 소리 | 1.5~3.5× baseFreq, 랜덤 딜레이 |

### 9.3 과일별 오디오 특성

| 과일 | baseFreq | resonance | 특징 |
|------|----------|-----------|------|
| apple | 2200 Hz | 0.7 | 중고음, 보통 울림 |
| orange | 1800 Hz | 0.5 | 중음, 짧은 울림 |
| grape | 3200 Hz | 0.9 | 고음, 긴 울림 |
| watermelon | 800 Hz | 0.3 | 저음, 짧은 울림 |
| strawberry | 2800 Hz | 0.8 | 고음, 긴 울림 |

### 9.4 두드리기 사운드
- **Glass Tap**: baseFreq × 1.2 Hz 사인파, 지속 80~120ms, 게인 0.15
- **Resonance Tail**: baseFreq 사인파, resonance 비례 감쇠 (100~400ms), 게인 0.08
- ±10% 랜덤 피치 변화

### 9.5 기타 사운드
- **파편 착지음**: 3000~7000 Hz 사인파, 지속 30~80ms
- **앰비언트**: 60 Hz 사인파, 게인 0.008 (상시)

## 10. 카메라 및 씬

- **카메라**: PerspectiveCamera, FOV 55, 초기 위치 (0, 1.5, 5)
- **OrbitControls**: 댐핑 0.05, 팬 비활성, 거리 2~10 제한, 슬라이스 중 비활성
- **조명**: Ambient(0x223344) + Directional×2 + PointLight(rim)
- **바닥**: 반사성 어두운 평면 (roughness 0.15, metalness 0.8)
- **안개**: FogExp2, density 0.08
- **톤매핑**: ACESFilmic, exposure 1.2

## 11. 과일 애니메이션

### 11.1 등장 애니메이션
- 스케일 0 → 1, 0.4초, ease-out 커브
- 살짝 위에서 떨어지듯 y +0.3 → 0 이동 동시 진행
- 등장 시 은은한 유리 공명음 재생 (baseFreq × 0.5, 300ms)

### 11.2 대기 상태
- Y축 회전: 초당 0.3 라디안
- 부유 효과: `sin(elapsed × 0.8) × 0.15`로 상하 이동
- 셰이더 uTime 유니폼 실시간 업데이트

## 14. 슬래시 트레일 시각 효과

- 2D 캔버스 오버레이 (포인터 이벤트 패스스루)
- 각 세그먼트의 나이(age)에 따라:
  - 300ms 이내만 렌더링
  - opacity: `(1 - age) × 0.8`
  - 선 굵기: `(1 - age) × 4 + 1`
- 글로우 레이어: 추가 8px 두께, opacity ×0.3
- 색상: rgba(255, 220, 220) / 글로우 rgba(255, 150, 150)
